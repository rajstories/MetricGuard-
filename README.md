# MetricGuard

**AI-powered metric consistency auditor.** Detects when the same business metric is defined differently across teams — the silent reason dashboards disagree and executives stop trusting the numbers.

## The Problem

A 10-person data team spending ~35% of its time reconciling metric definitions costs ~**$525K/year** in lost productivity. The root cause is almost always the same: Finance, Sales, Marketing, and Product each build their own version of "revenue" or "MAU" — same name, different SQL, different filters, different time windows — and nobody owns the canonical definition.

MetricGuard finds every one of those conflicts automatically.

## How It Works

```
metric_definitions.json          governed glossary
        │                               │
        ▼                               ▼
  [ Embedding ]              [ RAG Retrieval ]
        │                               │
        ▼                               │
  [ Cosine Similarity ]                 │
        │                               │
        ▼                               ▼
  [ Conflict Detection ] ──► [ LLM Agent (Claude Opus 4.7) ]
        │                               │
        └──────────────┬────────────────┘
                       ▼
              output/results.json
              output/dashboard.html
```

| Layer | File | What it does |
|---|---|---|
| Semantic Engine | `src/engine.py` | Embeds every metric definition as a vector, groups by cosine similarity, detects where same-meaning metrics have different logic |
| GenAI Layer | `src/genai.py` | RAG-retrieves the official glossary entry, calls Claude Opus 4.7 (adaptive thinking + prompt caching) to recommend the canonical fix |
| Report | `src/report.py` | Runs the full pipeline, writes `results.json` + `data.js`, prints headline KPIs |

## Results on Demo Dataset

| Metric | Value |
|---|---|
| Definitions scanned | 12 |
| In conflict | 8 **(66.7%)** |
| Conflict groups | 3 |
| Teams affected | 6 |
| Peak trust-risk score | 85 / 100 |

Conflicts detected: **Revenue** (refund handling + filter logic across Finance/Sales/Marketing), **MAU** (rolling-30d vs calendar-month + login-only filter across Product/Growth/Data), **AOV** (refund subtraction + completed-order filter across Growth/Finance).

## Quick Start

```bash
# Install dependencies
make install

# Run the full pipeline
make run

# Open the dashboard
open output/dashboard.html
```

Or step by step:

```bash
make engine   # semantic conflict detection
make genai    # RAG + LLM recommendations
make report   # write output/results.json and output/data.js
```

Set `ANTHROPIC_API_KEY` for live Claude recommendations. Without a key the pipeline uses pre-computed recommendations — the dashboard looks identical either way.

---

## Tech Stack

- **Embeddings** — `sentence-transformers` (all-MiniLM-L6-v2); TF-IDF offline fallback
- **Vector similarity** — cosine similarity via NumPy matrix multiply (same principle as Pinecone/FAISS at scale)
- **RAG retrieval** — same embedding + similarity search over a governed glossary
- **LLM** — Claude Opus 4.7 via Anthropic SDK with adaptive thinking and prompt caching
- **Dashboard** — self-contained HTML (no framework, no server needed)

## Extending It

Add your own metrics to `data/metric_definitions.json` (fields: `id`, `team`, `metric_name`, `description`, `sql`, and optionally `filters`, `includes_refunds`, `time_grain`). Add concepts to `data/glossary.json`. Then `make report`.
